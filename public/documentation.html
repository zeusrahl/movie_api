Documentation | Movie API

Intro Paragraph

This project was to build an API to help organize a set of data about movies. Mainly, it will revolve around users interacting movie names, directors, and genres. Additionally, users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request Method URL Response
Call the homepage GET '/' Returns with a welcome message: 'We are working with our Movie Database!'
List all movies to users GET /movies List of all available movies with image and title
Return data of requested movie GET /movies/:Title A JSON object with movie title, cover art, description, genre, featured film, director and actors
  • title: 'The Princess Bride',
  • description: "A fairy tale story about a beautiful young woman and her true love. They must battle the evils of their land and overcome the obsticles that keep them seperated.",
  • genre:
    • Name: 'Romantic Comedy'
    • Description: 'A story where the protagonist go through a series of comedic trials before finally falling in love.'
  • director: Rob Reiner{
    • bio: "Reiner started acting in the 1960's. In the 1980's he began a successful directing career, directing successful movies such as This is Spinal Tap, When Harry Met Sally, and A Few Good Men.",
    • birth: March 6, 1947,
    • death: Not applicable ,
    },
  • image: https://en.wikipedia.org/wiki/The_Princess_Bride_(film)#/media/File:Princess_bride.jpg
List of movies by genre GET /movies/genre/:Name Returns a list of movies that satisfy the requested genre
Return director information GET /directors/:Name Gives information about requested director with respect to their bio, birth day, death year (if deceased), and featured work:
  • director: Rob Reiner{
    • bio: "Reiner started acting in the 1960's. In the 1980's he began a successful directing career, directing successful movies such as This is Spinal Tap, When Harry Met Sally, and A Few Good Men.",
    • birth: March 6, 1947,
    • death: Not applicable ,
    },
Register a new user POST /users Add a new user to be registered. Returned data will result with a user that already exists, or with the following array:
  • Username: user1
  • Password: password1
  • Email: madeupemail1@gmail.com
  • Birthday: 01/01/2001
Pull up a list of users GET /users Pulls up all Users data
Pull a single user GET /users/:Username Pulls up all Users data, including an array of their favorite movies
Update a username PUT /users/:Username Update the username of a user
Users add movie to their favorites List POST /users/:Username/movies/:MovieID Add a single movie to User's list of favorite movies
Delete movie from user's List DELETE /users/:Username/movies/:MovieID A confirmation that a movie was deleted from User's favorite list
Deregister an account DELETE /users/:Username Username will be deleted from data
User login POST /login This will give a user a login token, allowing them to perform all the above operations
